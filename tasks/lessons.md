- Piping build output (`npm run package | tail -1`) masks failures: the pipe exits 0 even when packaging dies, so a stale dist gets installed. Always verify the installed bundle content (grep a fresh marker string in app.asar) before declaring shipped.
- node_modules corrupted by the interrupted overnight session (bad semver + http2-wrapper). Fix: rm -rf node_modules package-lock.json && npm install.
- Users downloading v1.0.0 got macOS "app is damaged": electron-packager breaks Electron's ad-hoc signature when it edits the bundle, and quarantined apps with invalid signatures show "damaged" (right-click Open does NOT bypass that variant). Fix: re-sign ad-hoc after packaging. Extra trap: signing fails inside ~/Documents because file-provider sync keeps re-stamping xattrs (com.apple.provenance); copy to /tmp with `ditto --noextattr` and sign there. Now automated as `npm run release-zip`.
- iCloud sync on ~/Documents evicts node_modules file contents over time (dataless files), corrupting the build toolchain repeatedly. Guard added: "prepackage" script requires key packages and runs `npm ci` if any are broken, so packaging self-heals. Always verify the installed app.asar contains a fresh marker string after any install.
- Shell cwd silently resets to ~/Documents between commands; a bare `git add -A` there once staged the ENTIRE parent Documents repo (timeout + stale index.lock + stray .gitignore). Always use `git -C <app path>` for repo operations, never rely on the persisted cwd.
